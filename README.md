# ResumeIQ AI — Intelligent Recruitment Platform

ResumeIQ AI is an agentic recruitment and resume screening platform designed to ingest candidate resumes (PDF, DOCX), extract profiles using natural language processing (NLP), and run advanced semantic evaluations against job description criteria. It integrates weighted matching metrics with executive AI feedback generated by Google Gemini.

---

## 🚀 Key Features

* **High-Fidelity Document Processing**: Secure, chunk-buffered document text extraction supporting PDF (PyMuPDF) and DOCX (python-docx) files. Implements file signature verification (magic bytes check) to prevent spoofing.
* **Granular Profile Parsing**: Powered by spaCy NER (Named Entity Recognition) and custom tech-skill regex taxonomies to extract contact details (email, phone, social links) and technical skills.
* **Semantic Compatibility Matching**: Uses Sentence Transformers (`all-MiniLM-L6-v2`) to generate 384-dimensional vector embeddings of resumes and job descriptions, calculating cosine similarities for deep semantic matching.
* **Neon PostgreSQL Serverless Integration**: Decoupled from heavy ORMs. Built entirely with raw, parameterized SQL queries via thread-safe `ThreadedConnectionPool` using the standard `psycopg2` driver. Includes automated table initialization on startup.
* **Executive AI Feedback**: Uses Google Gemini to generate three-bullet executive summaries detailing candidate suitability, highlighting strengths and potential skill gaps.
* **Developer Verification Dashboard**: A Next.js 15 app designed for modular API testing. Features live status monitors, document ingestion preview, and detailed ATS compatibility dashboards.

---

## 🛠 Tech Stack

* **Backend**: FastAPI, Python 3.12+, Psycopg2 (raw SQL), spaCy (NLP), Sentence Transformers (embeddings), PyMuPDF (PDF), python-docx (Word), Google Gemini (Generative AI).
* **Frontend**: Next.js 15, React, Tailwind CSS.
* **Database**: Neon Serverless PostgreSQL.

## ⚙ Startup Instructions

### 1. Backend Server Setup

Navigate into the backend folder, configure environments, and spin up the Uvicorn dev server:

```powershell
cd backend

# Activate Virtual Environment
.\venv\Scripts\activate

# Install Dependencies
pip install -r requirements.txt

# Start the Backend Server (Auto-builds schema tables on boot)
python main.py
```
*Backend runs live at: **http://localhost:8000** (interactive swagger docs available at `/docs`).*

### 2. Frontend Dashboard Setup

Navigate into the frontend folder and boot up the development server:

```bash
cd frontend

# Install Dependencies
npm install

# Start Dev Server (webpack-fallback enabled for App Control compatibility)
npm run dev
```
*Frontend runs live at: **http://localhost:3000**.*
